# Store and README Assets

This folder holds image assets used by the GitHub README and prepared for store submission. Runtime extension icons live in `src/assets/icons/` and are generated from `src/assets/icon-source.png` via `npm run icons`.

## Files here

- `re-phraser-icon-128.png` - 128x128 icon used as the README hero image and suitable as the store icon reference. Regenerated by `npm run icons`.

## Required icon sizes (runtime, in `src/assets/icons/`)

| Size | File | Used for |
|---|---|---|
| 16x16 | `icon16.png` | toolbar / favicon-scale |
| 32x32 | `icon32.png` | toolbar, in-page toolbar logo |
| 48x48 | `icon48.png` | extensions management page, options header |
| 128x128 | `icon128.png` | store listing and installation |

All are produced from the single source artwork; do not hand-edit the generated files.

## Screenshots (add before submission)

Real screenshots are **not** included in this repository yet. Capture them from the running extension and place them under `docs/assets/screenshots/` before store submission.

Recommended size: **1280x800** (works for both Chrome Web Store and Microsoft Edge Add-ons; both also accept 640x400).

Required shots (see `../store/screenshot-plan.md`):

1. Clean setup screen
2. Three-mode toolbar (Quick / Normal / Formal)
3. Suggestion preview with Apply / Cancel
4. Recovery card (missing URL or unreachable chat)
5. Settings screen

**Important:** screenshots must be captured from the real extension. Do not use mockups or fabricated images, and remove any sensitive text before saving.
